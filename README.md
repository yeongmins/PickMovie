# 🎬 PickMovie

**“뭐 볼지 1시간 고민하는 사람”을 위한, 취향 기반 영화/TV 추천 & AI 검색 플랫폼**

PickMovie는 **TMDB(The Movie Database)** 데이터를 기반으로 영화/TV 콘텐츠를 탐색하고,  
핵심 기능인 **Picky(AI 하이브리드 검색)** 로 사용자의 자연어 입력을 **TMDB 검색/필터 요청으로 변환**해 더 “의도에 가까운” 결과를 뽑아주는 프로젝트입니다.

> ✅ 목표: “디즈니 분위기 애니”, “지브리 감성”, “한국인이 좋아하는 로맨스” 같은 **자연어 검색**을  
> TMDB 결과(그냥 인기작 나열)에서 끝내지 않고 **의미 기반으로 보정**해 추천 품질을 끌어올리기

---

## ✨ 핵심 기능

### 1) 메인 탐색 (Browse)
- TMDB 기반 **인기 / 평점 / 현재 상영(또는 방영) / 트렌딩** 콘텐츠 탐색
- 통일된 **Content Card UI**
  - 영화/TV/애니 구분 뱃지
  - 현재 상영/방영 여부
  - 연령등급(가능한 경우)
  - OTT 시청 가능 플랫폼(가능한 경우)
  - 찜(즐겨찾기)

### 2) Picky (AI 하이브리드 검색)
자연어 입력을 “그대로 TMDB에 검색”하는 게 아니라, 아래 파이프라인으로 동작합니다.

**Picky 동작 방식(요약)**  
1. 사용자 입력: `“지브리 감성 애니 추천”`  
2. AI(Gemini)가 의도를 해석 → **JSON 결과** 생성  
   - 예: 키워드/장르/국가/시대/분위기/필터/가중치 등  
3. TMDB API 호출(검색/발견/상세/제공처/등급 등)  
4. 결과 후처리
   - Picky Lexicon(키워드 사전/브랜드/스튜디오/동의어)로 의도 보정
   - 필요한 경우 **매칭 점수(matchScore)** 계산/정렬(선택)

> ✅ “지브리” 검색 시 제목에 ‘지브리’만 들어간 작품이 나오는 문제를 줄이기 위해  
> **Lexicon + AI JSON 변환 + 후처리** 구조로 개선하는 방향을 지향합니다.

### 3) 찜 / 플레이리스트
- 콘텐츠를 찜하고 모아서 플레이리스트로 관리
- (현재 DB 미연동 환경에서는) 기본적으로 **클라이언트 저장소 기반**으로 동작할 수 있으며,
  추후 **DB 연동**으로 확장 가능한 구조를 목표로 합니다.

## 🔎 Picky 품질을 올리기 위한 설계 포인트
1) Lexicon(사전) 기반 의도 보정

브랜드/스튜디오/동의어/한글 표기 차이 등을 정규화

예: “지브리” → “Studio Ghibli”, “디즈니” → “Disney”

검색 쿼리/필터에 반영하거나, 결과 후처리에 사용

2) AI는 “추천 생성기”가 아니라 “필터/쿼리 변환기”

사용자 입력을 “영화 제목 리스트”로 직접 뱉게 하기보다,

TMDB에 던질 수 있는 구조화된 JSON을 반환하도록 프롬프트를 설계

결과가 틀리면: AI 응답(JSON) 로깅 → 규칙 보강(Lexicon/가중치/제약조건)

3) TMDB 한계 대응

TMDB에 없는 정보는 가져올 수 없으므로,

“한국 사용자 선호 데이터” 같은 별도 데이터는 추후 확장 과제로 분리

(예: 별도 랭킹/리뷰/메타데이터 DB 구축, 캐싱/수집 파이프라인 등)
